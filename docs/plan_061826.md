# Plan — Full TG68K core convergence: MacLC ⇄ upstream Minimig `030_mmu2`

**Date:** 2026-06-18 · **Branch to do the work on:** new `kernel-sync-030mmu2` (off `030_LCii_rebased` @ `3194397`)
**Goal (user, 2026-06-18):** bring MacLC's TG68K CPU core *into sync with — ideally cloned from —* upstream
Minimig `030_mmu2`, so the two cores can be co-developed and fixes flow both ways via PRs. Accept that this
is long; the payoff is a genuinely **shared CPU core**.

This plan supersedes the "adopt-their-kernel + re-add ours" framing in
`docs/handoff_kernel_merge_030mmu2_2026-06-18.md` — that handoff's feature claims were **measurably wrong**
(see §1). It also fulfils that handoff's request for a classified findings doc (§1 is the classification).

## 0. End state we are driving toward

**Target:** the **5 shared core VHDL files** are byte-identical between MacLC and Minimig `apolkosnik/030_mmu2`:
`TG68K_Pack.vhd`, `TG68K_ALU.vhd`, `TG68K_PMMU_030.vhd`, `TG68K_Cache_030.vhd`, `TG68KdotC_Kernel.vhd`.

- Mac-essential logic that upstream lacks is **upstreamed via PR** (so the shared file serves both), OR proven
  obsolete and dropped. We do **not** want a permanent local fork of any core file.
- The **integration layer stays per-project** and is *out of scope* for byte-identity:
  - MacLC: `tg68k.v` (sim **and** FPGA wrapper) + `TG68K.vhd` (reference-only, see §1.6).
  - Minimig: `cpu_wrapper.v` + `TG68K_CacheCtrl_030.vhd` (Amiga/Zorro integration — **not** core; exclude).
- Each convergence step must keep the **LC II boot at the desktop checkerboard (F399)**. That is the gate.

**Already done (do not redo):** `TG68K_PMMU_030.vhd` is byte-identical across MacLC/MacIIvi/Minimig. The
bsr.w/bsr.l PMMU-stall fix is in all kernels (MacLC `c8895d8`, Minimig `d21cd6c`, PR apolkosnik#3 open).

## 1. Corrected divergence map (the classification / findings)

Method: extracted Minimig pristine files to `/tmp/minimig_tg68k/` via
`git -C /Users/dani/repos/Minimig-AGA_MiSTer-danifunker show apolkosnik/030_mmu2:rtl/tg68k/<f>.vhd`, then
diffed feature-by-feature (raw line counts are inflated by debug ports, comment rewording, and
sensitivity-list reordering — feature greps are the real signal).

**Headline:** the two kernels share a common ancestor and were co-developed. They are FAR more converged than
"~290 vs ~576 lines" implies. The true delta is a small, well-bounded set of features.

### 1.1 `TG68K_PMMU_030.vhd` — ✅ byte-identical. Nothing to do.

### 1.2 `TG68K_Pack.vhd` — 1 line. (G)
Minimig adds `rte_mmu_replay` to the `micro_states` enum (used by the RTE-replay feature, §1.5). Trivial add.

### 1.3 `TG68K_ALU.vhd` — divergent implementation of the **same** fix. (G) — needs a decision
Both implement the DIVS.W overflow **N-flag** correctness fix (N must come from the *absolute* trial
quotient's low byte, not the sign-adjusted byte). They differ in how:
- **MacLC** recomputes `aquot := dividend_abs / divisor_abs` *inside* `divs_overflow_flags_68020` and reads
  `aquot(7:0)`.
- **Minimig** passes `quotient_abs_low8 = div_reg(7:0)` as a 3rd function arg and reads that.
**Action:** prove functional equivalence, then converge to **Minimig's** signature (so the file matches). If
MacLC's is actually more correct, upstream that instead. Either way: one shared impl. Verify with a DIVS.W
overflow test vector (negative divisor) — N-flag must match a 68030.

### 1.4 `TG68K_Cache_030.vhd` — ~15 generic blocks, ONE coherent design decision. (G, FPGA-only) — **highest risk**
Minimig switched the 68030 caches from **physical**-indexed/tagged → **logical**-indexed/tagged with
**FC-qualified tags** (new `i_fc`/`d_fc` ports), **removed write-allocate**, **relaxed cache-inhibit** (CI
blocks allocation only, not existing hits), and made CACR line-invalidate **unconditional**. MacLC keeps
physical tagging + write-allocate.
- **Why this is the risky one:** physical tagging sidesteps PMMU logical-address cache aliasing — exactly the
  area `f605e44`/`3194397` just stabilized for the LC II. Logical caching is more 68030-faithful but changes
  how lines key under active translation.
- **The sim CANNOT validate this** — `tg68k.v` runs the kernel **uncached** (cache not instantiated). Only
  FPGA exercises it. Requires real-HW validation against MAME.
- Needs `i_fc`/`d_fc` wired at the instantiation site (`tg68k.v` + `TG68K.vhd`).
- **No Mac-specific and no Amiga-specific gating** in the file — it's the same generic cache, opposite
  trade-offs. (Full per-block table: agent classification, archived in this plan's commit message / below.)

### 1.5 `TG68KdotC_Kernel.vhd` — the substance
| Feature (grep marker) | MacLC | Minimig | Class | Direction |
|---|---:|---:|---|---|
| `mmu_fetch_grace` / `mmu_grace_page` / `mmu_grace_suppress` | 7/5/4 | 0/0/0 | **M→G?** | MacLC-only; **upstream or prove obsolete** |
| `rte_fmt_a_replay_needed` / `_size` / `rte_fmt_a_ssw` | 0 | 7/9/12 | **G** | **pull into MacLC** |
| `rte_mmu_replay` (micro-state) | 0 | 6 | **G** | pull |
| `rte_format_b` (long MMU bus-fault frame fixup) | 0 | 9 | **G** | pull |
| `pmmu_pending_flags` | 0 | 4 | **G** | pull |
| `berr_exception_active` (double-fault detect) | 11 | 21 | **G** | pull (extend) |
| `rte_mmu_fix` (RTE frame-$B fixup family) | 63 | 94 | **G** | pull (extend) |
| `pmmu_fault_dispatched` / `_was_cleared` | 8/8 | 14/16 | **G** | pull (extend) |
| `pmove_dn*`, `pmmu_ptest_a`, `berr_ssw`, `pmmu_dbl` | ✓ | ✓ | shared | none |
| Entity **debug** ports | OP1out/OP2out | `pmmu_fault_*`, `USP/MSP/ISP`, `rte_mmu_fix_*`, `pmmu_pending_flags`, … | **S** | adopt Minimig's set |

- **MacLC-only `mmu_fetch_grace`** = "LC II boot bug #3": at the cycle `pmove (TC)` enables the MMU, in-flight
  **prefetched** instruction fetches must stay identity-mapped until the CPU branches out of the current page
  (else they translate mid-stream and fault). Tied to the `addr_out` / `pmmu_req` path
  (`TG68KdotC_Kernel.vhd:1064-1127, 8766`). **Genuinely load-bearing for the LC II boot.** It is *not*
  intrinsically Mac-specific — it's a TG68 prefetch-vs-MMU-enable hazard any 68030 core faces → strong
  **upstream-PR** candidate so the file stays shared.
- **Minimig-only RTE/MMU-fault family** is a coherent, more-complete fault-recovery implementation: replay a
  faulted data **write** on RTE (format $A), fix up the long MMU bus-fault frame (format $B), track
  pending/dispatched/cleared fault flags, detect double bus faults. The LC II currently boots *without* these
  → they are **not boot-critical**, but they're the upstream advance worth having (robustness for VM/fault
  workloads + convergence). **Note:** Minimig's additions are *not* purely additive — they edit the shared
  `rte5` state (`Kernel.vhd:7486+`) and the berr-frame path, so porting needs care (see §3 Phase D).

### 1.6 `TG68K.vhd` — **reference-only in MacLC** (inert). (S + one G)
`TG68K.qip:5-9` states TG68K.vhd is compiled by **neither** sim nor FPGA (both use `tg68k.v`). Diffs are the
debug-port surface + the same `i_fc`/`d_fc`→cache plumbing as §1.4. **Zero behavioral effect today.** Sync it
last, for cleanliness only.

### 1.7 Minimig extras NOT part of the core (exclude)
`TG68K_CacheCtrl_030.vhd` (Zorro/Fast-RAM + Amiga-walker-coupled cache controller) and `cpu_wrapper.v` are
Minimig's **Amiga integration layer**, instantiated only by `cpu_wrapper.v`. MacLC's portable cache is the
already-present `TG68K_Cache_030.vhd`. **Do not import these.**

## 2. Strategy: incremental convergence + bisection — NOT a wholesale swap

**Evidence base.** The prior wholesale swap (kernel+ALU+Cache+Pack) hung in **pre-MMU POST at `$A45Exx`**
(flat-grey screenshot). Decisive deductions:
1. The **sim is uncached**, so the cache swap *cannot* have caused that sim regression → the culprit is in the
   **kernel, ALU, or Pack**.
2. `mmu_fetch_grace` only acts **at/after MMU-enable**, which is *after* pre-MMU POST → the regression is **not**
   "missing grace." There is an **unidentified Minimig change that breaks a pre-MMU execution path** for the Mac.

Therefore: converge in **small, individually boot-tested steps**, newest-diff-isolated, so that the first step
which regresses identifies the offending change by bisection. A wholesale swap hides which change broke it.

**Two directions that together produce "in sync":**
- **Pull into MacLC:** Pack enum, ALU DIVS variant, the kernel RTE/MMU-fault family, debug ports, (gated) cache.
- **Push upstream from MacLC:** `mmu_fetch_grace` (if still needed) + the DIVS impl if MacLC's proves better.

## 3. Phased implementation (each phase: edit → ghdl gate → regen → build → boot → commit)

> Per-iteration cost: `convert_to_verilog.sh` ~30 s · Verilator `-Os` build ~8 min · boot test ~15–20 min.
> Minimize full builds: always pass the **fast ghdl `--synth` gate** (§4.1) before spending a build.
> Commit after every green phase so bisection has clean anchors.

### Phase A — Baseline + harness (no logic change)
1. New branch `kernel-sync-030mmu2` off `3194397`.
2. Re-confirm the **baseline** boots: build current tree, `--screenshot 399 --stop-at-frame 400`, verify HB
   progression `$A45Exx→$A07A5A→$A3Cxxx→$A14xxx→$A06F08` and a **checkerboard** screenshot. This is the
   reference image/log every later phase is compared against. (Vemu binary already exists; rebuild to be sure.)
3. Snapshot `/tmp/minimig_tg68k/` extraction commands into the branch (a `tools/` note) for reproducibility.

### Phase B — Debug-port convergence (S; zero intended logic change) — *de-noise the diff first*
Adopt Minimig's kernel **entity debug-port set** (drop `debug_OP1out/OP2out`; add `debug_pmmu_fault_*`,
`debug_USP/MSP/ISP`, `debug_a7_is_msp`, `debug_interrupt_mode`, `debug_rte_saved_mbit`, `debug_rte_format_word`,
`debug_rte_mmu_fix_*`, `debug_pmmu_pending_flags`, `debug_berr_exception_active`). These connect to internal
signals that arrive with the §1.5 features — so **Phase B is folded into Phase D** in practice (you can't add a
debug port for a signal that doesn't exist yet). Rule: whenever a pulled feature lands, bring its debug ports
with it verbatim. `tg68k.v` currently connects **none** of OP1out/OP2out, so dropping them is free; new ports
are left `open` unless a probe wants them. **Outcome:** the only non-feature diff left is comments/whitespace.

### Phase C — Pack enum + ALU DIVS (G; small, generic)
1. `TG68K_Pack.vhd`: add `rte_mmu_replay` to `micro_states`. (Harmless until the kernel uses it.)
2. `TG68K_ALU.vhd`: converge `divs_overflow_flags_68020` to Minimig's signature (`quotient_abs_low8` arg).
   Before building: hand-check equivalence on a negative-divisor overflow vector. ghdl `--synth` gate, then a
   build+boot (DIVS.W is exercised constantly — boot is a real test here).
3. Commit "Pack+ALU: converge rte_mmu_replay enum + DIVS N-flag impl to upstream."

### Phase D — Kernel RTE/MMU-fault family (G; the substance; **bisection-critical**)
Pull Minimig's fault-recovery family in the **smallest coherent sub-steps**, boot-testing each. Suggested order
(each is its own commit + boot test):
- **D1.** Additive signals + `pmmu_pending_flags` + the `berr_exception_active` extension (state plumbing that
  doesn't change existing transitions). Boot.
- **D2.** `rte_format_b` long MMU bus-fault frame fixup + extended `rte_mmu_fix` family. Boot.
- **D3.** `rte_fmt_a_replay_*` + the `rte_mmu_replay` micro-state, including the edits to the **shared `rte5`
  state** (`Kernel.vhd:7486+`). This is the most likely boot-regressor (touches a live path). Boot.
- **D4.** `pmmu_fault_dispatched/_was_cleared` extension (double-fault detect). Boot.
- After each: `convert_to_verilog.sh` → ghdl gate → build → boot. **If a sub-step regresses pre-MMU POST**, you
  have isolated the offending hunk — diff that sub-step against Minimig line-by-line and find the shared-path
  edit that the Mac depends on (this is the "unidentified pre-MMU breaker" from §2).
- **Validation beyond boot:** the boot doesn't trigger MMU bus faults, so "feature works" can't be proven by
  boot alone. Confirm (a) ghdl `--synth` clean, (b) boot unregressed, (c) the ported logic is **textually
  identical** to Minimig's for the shared blocks. Deep fault-path validation (RTE-from-fault) needs a
  VM-enabled OS or a crafted test under MAME oracle — track as a follow-up (§4.4), not a Phase-D gate.

### Phase E — `mmu_fetch_grace` decision (the one MacLC-only core block)
1. **Test obsolescence:** with the PMMU early-term fix (`f605e44`) in place, build a variant with
   `mmu_fetch_grace` **removed** (grace_suppress hard-0) and boot. The handoff explicitly flags it as *possibly*
   an obsolete workaround.
   - **If the LC II still boots to checkerboard** → drop it. MacLC's kernel converges to Minimig automatically;
     no upstream PR needed for it.
   - **If boot regresses** → it is load-bearing. Keep it, and **prepare an upstream PR** framing it as a generic
     68030 *prefetch-vs-MMU-enable* hazard fix (not Mac-specific) so Minimig adopts it and the file stays shared.
     Until merged upstream, this is the **one** allowed local delta; document it at the top of the file.
2. This phase is where "byte-identical kernel" is won or deferred to the PR landing.

### Phase F — Cache (G; FPGA-only; **defer behind HW validation**)
1. Adopt Minimig's logical-tag + FC-qualified cache (`i_fc`/`d_fc`), drop write-allocate, relax CI, unconditional
   invalidate. Wire `i_fc`/`d_fc` from `fc_internal` in `tg68k.v` (and `TG68K.vhd` for the reference file).
2. **Sim cannot validate** — boot test will pass trivially because the sim is uncached. So this phase is gated on
   **real FPGA boot of the LC II + a MAME framebuffer/PC-stream oracle** with caches enabled, not on the sim.
3. **Recommendation:** stage this as the **last** core change and keep a clean revert. If FPGA boot regresses
   post-MMU, the physical-vs-logical tagging interaction with the PMMU is the suspect (it's the exact aliasing
   property MacLC's physical tags avoid). Acceptable outcome: if logical caching destabilizes the LC II on HW,
   keep MacLC's physical cache as a **documented, upstream-proposed** deliberate divergence rather than break the
   board — i.e. converge everything else and treat the cache as a separate negotiated PR.

### Phase G — `TG68K.vhd` reference-file sync (S; inert)
Mirror the debug-port surface + `i_fc`/`d_fc` plumbing so the reference file matches. No build impact.

### Phase H — Prove byte-identity
`for f in Pack ALU PMMU_030 Cache_030 Kernel; do diff rtl/tg68k/TG68K_$f.vhd /tmp/minimig_tg68k/TG68K_$f.vhd; done`
→ empty (modulo any Phase-E/F deltas explicitly retained pending upstream PRs). Regenerate `.v` once more, final
build + boot, screenshot archived.

### Phase I — Upstream PRs (the "push" direction)
- PR #A: `mmu_fetch_grace` as a generic 68030 prefetch/MMU-enable hazard fix (if Phase E kept it).
- PR #B: DIVS N-flag impl, if MacLC's proved more correct (else none — we adopted theirs).
- PR #C (optional): MacLC's physical-cache option as a config, if Phase F kept it.
- Target `danifunker:030_mmu2_fixes → apolkosnik:030_mmu2` (same route as the open bsr.w PR #3).

### Phase J — Propagate to MacIIvi
After MacLC core is green + (mostly) byte-identical, copy the 5 core files (+ regenerated `.v`, + any `tg68k.v`
debug/cache wiring) to `/Users/dani/repos/MacIIvi_MiSTer`, build/boot that core's own test, commit. Keep all
three repos' cores identical.

## 4. Testing methodology

### 4.1 Fast ghdl elaboration gate (seconds — run before EVERY build)
```bash
W=$(mktemp -d); for f in TG68K_Pack TG68K_ALU TG68K_PMMU_030 TG68K_Cache_030 TG68KdotC_Kernel; do \
  ghdl -a -fsynopsys -fexplicit --workdir=$W rtl/tg68k/$f.vhd; done
ghdl --synth -fsynopsys -fexplicit --latches --workdir=$W TG68KdotC_Kernel   # exit 0 = clean (--latches REQUIRED)
```

### 4.2 Regenerate Verilog + build
```bash
rtl/tg68k/convert_to_verilog.sh          # ~30s; analyzes Pack→ALU→PMMU→Cache→Kernel, inlines PMMU into the .v
cd verilator
make obj_dir/Vemu.cpp && (cd obj_dir && rm -f *.o *.gch && make OPT_FAST=-Os OPT_SLOW=-Os -f Vemu.mk)  # ~8 min
# (plain `make` = -O0 = 15x slow / looks hung; `make fast` crashes clang — always -Os)
```

### 4.3 Boot regression gate (the must-hold test, run ONCE per phase, then analyze logs)
```bash
./obj_dir/Vemu --headless --no-cpu-trace --heartbeat --rom ../releases/boot0-fastmem.rom \
  --screenshot 399 --stop-at-frame 400 > /tmp/boot.log 2>&1
grep '\[HB\]' /tmp/boot.log | tail   # PASS: PCs climb $A45Exx→$A07A5A→$A3Cxxx→$A14xxx→$A06F08
                                     # FAIL: stuck looping in $A45Exx (pre-MMU POST)
# screenshot_frame_0399.png: PASS = desktop CHECKERBOARD; FAIL = FLAT uniform grey
```
Keep `verilator/Makefile`'s `PMMU_TRACE`/`A30_TRACE` defines **commented** for normal builds (enabling them makes
`tg68k.v` connect MacLC-only kernel `debug_*` ports — a build-break signal, not a goal).

### 4.4 Deep fault-path validation (follow-up, not a per-phase gate)
The boot path does not raise MMU bus faults, so the RTE-replay/format-$B features can only be *boot-unregressed*,
not *exercised*, by the boot test. To actually exercise them: run a System 7 image with **virtual memory on** (or
a crafted PTEST/fault test) under both the core and MAME `maclc2`, and diff the maincpu PC stream around the fault
(`docs/mame_compare.md`, `verilator/mame/run_mame_maclc2.sh`, the `mmu_*.lua`/`pt_root10.lua` probes). Schedule
this once the family is in and boot-green.

### 4.5 FPGA validation (required for Phase F cache only)
Quartus build → DE10-Nano → LC II boot with caches enabled; compare framebuffer + behavior to MAME. The sim
**cannot** cover the cache.

## 5. Risk register
| Risk | Likelihood | Mitigation |
|---|---|---|
| A pulled kernel hunk edits a shared pre-MMU path and hangs POST (the §2 unidentified breaker) | **High** | Phase D sub-steps + per-step boot test → bisect to the exact hunk; diff that block vs Minimig |
| `mmu_fetch_grace` still needed → blocks byte-identity | Medium | Phase E test; if needed, upstream PR; allow 1 documented local delta until merged |
| Logical cache destabilizes LC II post-MMU on HW (sim can't catch) | Medium | Phase F last + revert path; keep physical cache as negotiated divergence if HW regresses |
| DIVS impl swap changes N-flag on some vector | Low | Equivalence check + boot (DIVS.W heavily exercised) |
| Debug-port reshuffle breaks `tg68k.v` binding | Low | tg68k.v connects none of the dropped ports; add new ones as `open` |
| Regen (`convert_to_verilog.sh`) drift vs hand VHDL | Low | ghdl `--synth` gate every step; never hand-edit the generated `.v` |
