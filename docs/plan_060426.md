## Goal

Bring this core's MiSTer framework (`sys/`) up to the current upstream
template, while preserving the carefully-tuned core behavior (boot, video,
Egret, RAM mapping) and our local fixes.

## Hard constraint: verification is Quartus-only

The Verilator sim builds `verilator/sim.v` (`module emu`), **not**
`sys_top.v` / the framework. **No framework change can be validated in
Verilator.** The only validation path is a **Quartus rebuild on the Windows
machine** (`/Volumes/Temp/mistercore/MacLC_MiSTer`) followed by an on-hardware
boot. Plan accordingly: every framework step is "blind" until that rebuild.

## Phase 1 — Inventory (already scouted; recorded here)

Files that differ (`sys/` ours vs template):

| File | Change | Risk |
|---|---|---|
| `osd.v` | 2 lines | trivial |
| `hps_io.sv` | 79 lines; **`HPS_BUS` width `[48:0]`→`[45:0]`**, new `F12KEYMOD` param | **HIGH (coupled interface)** |
| `sys_top.v` | 28 lines; audio boost, `HDMI_TX_INT`→`gp_in`, `N_BURST`, HPS cmds `0x2B/2F/3E/44`; reverts `.mode` to 4-bit | MEDIUM |
| `ascal.vhd` | full new version (~6k-line diff); `mode` still 5-bit | LOW (drop-in) |
| `audio_out.v` → `audio_out.sv` | renamed + new `.boost` port | MEDIUM (qip edits) |
| `emu_ports.vh` | **new** include file (emu port list) | optional |

**Compatibility confirmed:**
- The `emu` *instantiation* port list in template `sys_top.v` is **identical**
  to ours (no port renames).
- Our `MacLC.sv` emu port set already contains every port the new framework
  references (LED_*, SDRAM_*, UART_*, HDMI_*, VGA_*, FB_*).
- The **only** emu-interface change required is `HPS_BUS [48:0]→[45:0]`.

**Critical coupling:** `HPS_BUS` width must match in all three:
`hps_io.sv` (module decl), `sys_top.v` (instantiations), and
`MacLC.sv` (emu port). Template is internally consistent at `[45:0]`
(it dropped the `fb_en`+scanlines bits from the bus). Updating must be
all-or-nothing across these three files.

## Phase 2 — Execute the update

Do as **one commit**, in this order:

2.1 **Drop-in files** (no interface change):
- Copy `../Template_MiSTer/sys/ascal.vhd` → `sys/ascal.vhd`
- Copy `../Template_MiSTer/sys/osd.v` → `sys/osd.v`

2.2 **audio_out rename:**
- `git rm sys/audio_out.v`; copy `../Template_MiSTer/sys/audio_out.sv` → `sys/audio_out.sv`
- Edit `sys/sys.qip:24`: `VERILOG_FILE ... audio_out.v` →
  `SYSTEMVERILOG_FILE ... audio_out.sv` (template already has this form — can
  copy template `sys.qip`).
- Grep `files.qip` / `MacLC.qsf` for any other `audio_out.v` reference (none
  found in scout, but re-verify).

2.3 **hps_io.sv:**
- Copy `../Template_MiSTer/sys/hps_io.sv` → `sys/hps_io.sv` (brings `[45:0]`
  bus + `F12KEYMOD`).

2.4 **sys_top.v:**
- Copy `../Template_MiSTer/sys/sys_top.v` → `sys/sys_top.v`, **then re-apply
  our local fix**: change `.mode ({~lowlat,...})` →
  `.mode ({1'b0,~lowlat,...})` (keeps the ascal 5-bit `mode` Critical Warning
  cleared; upstream left it 4-bit).

2.5 **MacLC.sv (emu top) — the one interface edit:**
- `inout [48:0] HPS_BUS,` → `inout [45:0] HPS_BUS,`
- Decide on `emu_ports.vh`: **recommended NO** for this pass — keep our inline
  port list to minimize change surface. (Adopting the include is a separate,
  optional cleanup once the build is proven.)

2.6 **Quartus message-suppression:** revisit `MacLC.srf` — the new framework may
emit different IDs. Re-run and re-curate after first build.
